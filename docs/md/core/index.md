# rainy-toolkit 核心库

rainy-toolkit是一个较有规模的基础设施集合库.

但rainy-toolkit也有自己的体系，它的核心库（core库）是所有rainy-toolkit模块的基础。几乎所有rainy-toolkit模块都依赖于核心库。例如，每个来自rainy-toolkit的模块中（除核心库外）总是会包含<rainy/core/core.hpp>头文件，表示对核心库的引用。

```cpp
#include <rainy/core/core.hpp>
```

而核心库中，通常自举了大量模块,并自动包含了下列的模块： 
 - 基础算法组件 
 - 类型特征库（类似于<type_traits>但是提供了更多的类型特征以供检测）
 - 基础实用库组件 
 - 平台抽象层
 - 原子指令集合
 - 少量基于汇编的封装
 - 生命周期注解
 - 异常拓展
 - array以及array_view
 - basic_poly多态
 - 必要的全局变量和平台宏 
 
虽然在正常使用中，不建议直接访问core命名空间。而是通过其它命名空间间接访问核心库的内容。这通常是考虑到核心库的内容实现部分比起其它模块更为底层、并且在后续中，核心库的部分底层代码有大概率会被重构（尽管整体上仍然很稳定）但是无论如何，访问core命名空间并不安全。除非你正在编写基于核心库的模块。

本文档中,则着重介绍核心库中，面向用户的API部分。在此之前,请确保你已经包含了核心库的头文件,然后再开始.

::: info
核心库是指rainy::core命名空间下的所有API。而在核心库中出现的其它命名空间,例如rainy::meta、rainy::platform等,则是核心库的一部分.但是它们被认为是核心库向外暴露的子模块.因此,它们的文档应当在各自的模块文档中查阅.而不是在核心库文档中查阅.
:::

此外,核心库core包含了以下子命名空间:

- builtin: 核心库的底层内建模块
- abi: 核心库的ABI相关模块,用于帮助模块的识别
- algorithm: 自建的基础算法组件
- pal: 平台抽象层
