## 概要

basic_any类是一个类似于std::any的容器。但是，存在显著区别。比如，在所有权机制方面，std::any使用的是传统的RAII策略，即资源的获取和释放与当前变量进行了绑定。而basic_any相反，它们被设计为一个混合的所有权设计的动态变量容器。

例如，在持有某个对象，并且明确basic_any负责其声明周期的时候，basic_any将会采用RAII策略，反之，如果basic_any持有了引用，这一状态被称为借用所有权，不会进行资源的清理。但是，无论是借用还是持有，这些所有权都可释放。持有所有权按RAII策略处理，借用所有权将会置空basic_any。

同时，类型信息并不基于RTTI，此外，basic_any有大量功能不同于std::any。例如，basic_any被用于访问对不确定的变量或引用进行运算符或访问操作。并且，在可能的情况下，还会执行跨类型、同类型引用的转换操作以方便适配操作。

通俗的说，basic_any是一个动态变量，其被用于灵活的管理不确定类型的对象。同时，也被用于简化大量的重复性代码。而std::any，则会更加倾向于轻量化的存储。

当然，其混合所有权也意味着更加灵活的访问资源。详情资料请参加末尾的所有权机制。不过简要的说，对于资源，必要的时候需通过接口判断其是否具有所有权。这一部分，由开发者所控制，具体实践由开发者决定。
