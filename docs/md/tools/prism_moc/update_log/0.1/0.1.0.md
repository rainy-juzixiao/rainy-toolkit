## 0.1.0

首个alpha技术测试版本，可生成一部分可用的基础注册代码，用于验证技术可行性质

给定示例

```cpp
class Foo {
};

class Bar {
};

class a {
public:
    RAINY_ENABLE_MOC();

    a() { }

    void foo() {

    }

    void foo(int, int) {

    }

    static void bar() {}
};
```

生成如下代码：

```cpp
#include <rainy/meta/reflection/registration.hpp>
#include "misc/example.cc"

using namespace rainy::meta::reflection;

RAINY_REFLECTION_REGISTRATION {
    registration::class_<a>("a")
        .method(rainy::utility::get_overloaded_func<a, a()>(a::)
        .method(rainy::utility::get_overloaded_func<a, void()>(&a::foo))
        .method(rainy::utility::get_overloaded_func<a, void(int, int)>(&a::foo))
        .method(rainy::utility::get_overloaded_func<a, int()>(&a::mem))
        .method(rainy::utility::get_overloaded_func<void()>(&a::bar));
}
```

主要缺陷：

- 无增量编译机制，每次执行生成器都会进行无用的解析和重新生成
- 仅支持单文件扫描单个类
- 由于技术测试特性，无法定制输入接口
- 界面简单
- 仅支持生成类的注册，无法识别类构造函数
- 头文件包括机制有缺陷
- 无法将错误进行line的校准
- 无法将名称注册到反射系统中导致编译错误
- 生成的代码不可用

目前确立的点：

- 使用blake3算法进行增量哈希识别
- 不考虑支持扫描源文件内的头文件，仅解析当前源文件进行codegen
- 制作构建系统的集成
- 考虑现代化Cli界面
